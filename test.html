CSC-492-Project

Problem: It can be difficult to plan and accurately track your fitness goals

Users: User 1

Name: Larry

Age: 32

Occupation: Accountant

Background: Lives in Lowell, Massachusetts Social Man Has a wife and three children. Studied at Baruch College in New York

Frustrations: Work Having to commute 90 minutes every day Gaining weight during covid

Bio: Larry lives with his wife Sarah, and his three children: Mike, Jackie, and Harrison. His passions are the outdoors; he enjoys hiking and camping particularly. He enjoys time with his friends and family. His favorite hobby is collecting baseballs and baseball cards. He loves watching red sox and patriots games and he has Red Sox season tickets. His favorite TV show is the Sopranos.

Problem Scenario:

Larry and his family love hiking but during covid Larry gained weight due to his local gym closing down. Larry has been struggling hiking with his family and has to cut hikes short because he can no longer handle long hikes and has been having difficulty establishing a routine working out. Larry uses the fitness calendar app to solve his routine difficulties. Larry establishes his daily workouts on the calendar so he will not forget, each detailing what he should be doing. Furthermore, Larry has adopted a better diet to help in his weight loss process and has developed a schedule using the app of what he is eating each day, monitoring his calorie intake as well.

Users: User 2

Name: Derek

Age: 29

Occupation: Kitchen Staff

Background:

  • - Lives in Greenville, South Carolina
  • - Lives on their own in an apartment outside the city
  • - Loves to boulder indoors and in nature.
  • - Fairly knowledgeable in regards to fitness for climbing

Frustrations:

  • - Fears getting injured by not getting proper rest
  • - Driven to grind out their workouts but again concerned with injuries
  • - Lack of organization with their workouts

Bio: Derek is a casual climber who focus on bouldering. He recently injured his shoulder by bouldering at a level beyond his own strength and is looking to gain practical strength to avoid further injuries. Derek has some difficulty organizing his workouts and targeting specific workouts and body parts. He often finds himself not really knowing what to focus on in the gym and typically rotates between the same 4 workouts. Derek is looking for a way to organize his workouts and track his progress and implement progressive overload.

Problem Scenario:

Derek has recently been able to get back into climbing after his shoulder injury. Derek is gearing up to go to the gym and needs to organize a workout. He loads up his profile on the website and looks at selects his the current day on the calendar. Derek looks up his prerecorded basic upper body routine from a few days ago. Reading the note he left on the workout last time, Derek was reminded that he wasn’t very challenged by the pull up portion. Using this information Derek is going to attempt to add 3 reps of pull-ups per set. He updates the workout and heads to the gym.

Achieving his goal and feeling challenged by the change in reps, Derek loads up his profile and checks off that he completed the workout. He then begins to plan a new workout for his next session.

Use Cases

  • 1. The system asks the user for their username
  • 2. User provides username
  • 3. The system displays the a the calendar with the users previous workouts noted
  • 4. The user clicks the previous date with a previous workout
  • 5. The system opens displays the given days workout and note left by user
  • 6. The user clicks the current day
  • 7. The system opens a notepad for the given day, allowing the user to enter their workout.
  • 8. The user enters their workout
  • 9. The user clicks the save
  • 10. The system saves the users workout.

Requirements

Requirement 1 - Functional

Description

The system should allow users to enter their workouts in any basic text format. This should allow users to bullet point and checkbox their workouts.

Rationale

The core of this app is to give users a convenient way to plan workouts.


Requirement 2 - Data

Description

The system should save the user’s entered workouts under some personal key. This information should be available to the user by clicking the date of the workout.

Rationale

This will allow the user to see their previous workouts and plan future ones accordingly.


Requirement 3 - Functional

Description

The systems main page calendar should display a title for days that workouts have previously been entered

Rationale

This will allow the user to easily see what days they have previously entered workouts, without having to remember.


Requirement 4 - Functional

Description

The system should allow the user to enter their target weight and timeframe in which they would like to achieve it.

Rationale

This information will give requirement #5 the data it needs to function


Requirement 5 - Functional

Description

The system should be able to calculate the caloric intake and what level of a caloric deficit the user must maintain in order to achieve their weight loss gains within their chosen timeframe.

Rationale

This provides the user with a rough plan to achieve their weight loss goals. Allowing them to make a judgement call on what the would like to do.


Requirement 6 - Data

Description

The system should be able to store the users target weight, timeframe and the calculations made using them.

Rationale

This will allow requirement # 7 to function


Requirement 7 - Functional

Description

The system should display the users target weight, timeframe and the systems calculations on the main calendar page.

Rationale

This will allow the user to see their goals anytime, further motivating and informing their actions.


Requirement 8 - Data

Description

The system must have access to a nutrition database such as the USDA api

Rationale

This information is vital in helping users maintain their caloric goals.


Requirement 9 - Functional

Description

The system should allow users to search for the meals they’ve eaten using requirement #8. The system should be able to allow the user to enter their meals or the calories of their meals if the meal was not found within the database.

Rationale

This will allow the system to calculate the total caloric intake of the user


Requirement 10 - Functional

Description

The system should be able to calculate the total caloric intake of the user and roughly determine if the user is at enough of a caloric deficit to reach their weight loss goals.

Rationale

This will allow the user to make changes, if necessary to either their diet and workouts or their weight loss goals.


Requirement 11 - Data

Description

The system should not only store custom workouts entered by the user but it should also have common workouts tailored to specific muscle group.

Rationale

This will allow the system to recommend workouts as seen in requirement #13


Requirement 12 - Functional

Description

The system should allow users to select a muscle group they would like to focus on.

Rationale

This will allow requirement # 13 to be possible


Requirement 13 - Data

Description

The system should recommend workouts to the user for a given day based on their chosen muscle group in requirement #12

Rationale

This will give users further guidance in choosing effective workouts to achieve their fitness goal whether that be weight loss or strength gains.
